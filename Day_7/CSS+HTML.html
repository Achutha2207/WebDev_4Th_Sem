Integrating HTML+CSS Can Be Done In 3 Ways

  • Inline
  • Internal
  • External
INLINE CSS
Usage Of Inline Css Is done by Inserting The STYLE attribute
to tags eg: <p style="color:green;"> <p>
INLINE CSS referes to as if we are adding style attribute
inline each line and to every-line to style the html code
INTERNAL CSS
Using the style tag inside the head tag to mention the
Styling for common elements in HTML eg:
{INSIDE HEAD TAG} <style> h1{color:grey;} </style>
here for all the h1 tag common color grey is applied
This is a massive advantage contrary to INLINE CSS
Where we need to add the style attribute to each and every
Element in the file to style it

But,in INTERNAL CSS we Have the advantage of
Heirarchicaly Styling [Styling for the same class of tags]
EXTERNAL CSS
We use the STYLING Independently And seperately.
We seperate CSS and HTML and treat them as seperate entities
Where this gives us the advantage of dividing the strengths of
HTML and CSS as a core language and use them as we need.
It also helps us seperate the UI part with the styling part
so that one can work on the elements and another person
can work on the styling part we generally use
<link rel="stylesheet" href="CSS_FILE_NAME.css">

Usage of EXTERNAL CSS is also benificial to the client as
he/she can link/unlink the css file / add another css file/
Test multiple stylesheets to evaluate which one of those files
best fits his/her needs of the Website's Front-End
